# Design: Single-Source Language Reference → Three Artifacts

## Variables

- **Goal:** one canonical language reference; three derived artifacts; no manual
  re-copy; no drift between docs, agent prompt, and the real op set.
- **Artifact model:** source = `public/docs/ecky-ir.md` (prose/examples) +
  `surface-reference` (op catalogue, already auto-derived from ops).
- **Variables:** what is "source" vs "generated"; the compression transform for
  the API prompt; where the API prompt artifact lives; how MCP + API share it;
  the drift check.
- **Decision:** below.
- **Rejected paths:** (a) keep three hand-maintained copies — the status quo,
  drifts; (b) make the book itself the verbatim system prompt — too long, has
  images, narrative tone wastes context; (c) hand-author the API prompt
  separately — forks again.
- **Proof plan:** golden-file test of the generated prompt; drift test
  (prompt op list == surface-reference == book appendix); a self-contained check
  (prompt references no MCP tool / no image).

## Source of truth

| Content | Source | Already auto? |
| --- | --- | --- |
| Concepts, grammar, worked examples, pitfalls | `public/docs/ecky-ir.md` | no — hand-authored prose |
| Op catalogue (names, signatures, per-backend support) | `surface-reference` (`ecky_language_surface`) | **yes** — derived from ops |

The book stays the human-authored prose source; the op table is **never**
hand-written into the prompt — it is injected from `surface-reference`, so adding
an op (as we did for `torus`, `slot`, …) updates the prompt automatically.

## The three artifacts

```
public/docs/ecky-ir.md  ─┐
                         ├─► build:book ─► 1. Book (EPUB/HTML, images, full prose)   [humans]
surface-reference (ops) ─┘              ├─► 2. MCP guides (ecky://guides/*)          [in-app MCP agent]
                                        └─► 3. API system prompt (compressed)        [API agent, no MCP]
```

All three are generated by one pipeline so the agent-facing text (2 and 3) and
the human book (1) share their language content.

## Compression transform (book md → API system prompt)

Deterministic projection, in order:

1. **Strip images** — remove every `![…](…)` line.
2. **Distill prose** — keep code blocks, rules/gotcha callouts, and the op/units
   tables; drop chapter narrative intros ("warm" framing). Drier is fine.
3. **Inject op catalogue** — append/insert the `surface-reference` op table
   (compact: name · signature · backends) instead of any hand-written op list.
4. **Prepend the API operating contract** (see below).
5. **Budget guard** — assert the result is under a configured token/char ceiling;
   fail the build if the distilled prompt overflows so it stays usable as a
   system prompt.

The transform is mechanical and tested with a golden file so changes are
reviewable.

## API-mode operating contract (prepended to the prompt)

Because the API agent has no MCP:

- You author **`.ecky` source only**. There are no tools to call and no guides to
  read — everything you need is in this prompt.
- All dimensions are millimetres; angles in degrees.
- On a failed request you receive the **compiler diagnostic**; fix the named
  cause and re-emit. Treat diagnostics as authoritative (e.g. native-only ops
  like `:created-by`, `:to-radius` rejected by build123d).
- Prefer geometry that renders on the default backend; respect the per-op
  backend support in the op table.

## Sharing between MCP and API

Introduce one `agent_language_reference()` builder that assembles: the API
operating contract + the distilled language body + the injected op catalogue.
- API mode uses its output directly as the system prompt.
- The MCP `ecky://guides/technical-system-prompt` (and language guide) serve the
  same body, so the in-app agent and the API agent read identical language rules.

## Drift check (CI)

A test asserts:
- every op named in the generated prompt exists in `surface-reference`, and
- every `surface-reference` op appears in the prompt's op table, and
- the book appendix op list matches `surface-reference`.

So a new op cannot ship without appearing in all three, and a removed/renamed op
cannot leave a stale mention behind.

## Resolved decisions

1. **Artifact home — committed `.md` under `docs/`, with a CI freshness gate.**
   The prompt defines agent behaviour, so its changes must be visible in diffs
   and reviewed (unlike the EPUB, a binary blob that is gitignored). To avoid a
   committed-generated file going stale, CI regenerates it and fails on any diff
   with the committed copy ("checked-in generated + freshness gate"). Bonus: an
   op addition shows up explicitly in review as a prompt diff.
2. **Budget ceiling — 8K tokens (~32K chars), configurable.** A dry distillation
   of the ~68 KB book (grammar + ~75-op catalogue + rules + pitfalls + contract,
   images and narrative stripped) fits comfortably under this. 8K is ~4% of a
   200K window — cheap per turn, leaves room for the conversation and the model's
   reasoning. The ceiling is a discipline signal: overflow means the distillation
   isn't tight enough, not that the limit should rise. Start at 8K; bump to 12K
   only if the complete reference genuinely cannot fit. The limit lives in config
   so it is one number to tune.
