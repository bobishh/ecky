# Delta for agent-language-reference

## ADDED Requirements

### Requirement: One source emits all three language artifacts

The system SHALL derive the human book, the MCP language guides, and the API
system prompt from a single language source (`public/docs/ecky-ir.md` for prose
plus the auto-derived `surface-reference` op catalogue), generated by one
pipeline.

#### Scenario: A language-source edit reaches all artifacts

- GIVEN an edit to the language source
- WHEN the book pipeline runs
- THEN the book, the MCP language body, and the API system prompt all reflect the edit
- AND no artifact requires a separate manual copy.

### Requirement: The API system prompt is self-contained and compressed

The system SHALL generate an API-mode system prompt that contains the full
language reference an agent needs to author `.ecky` source, with images removed
and prose distilled, and that references no MCP tool — because the API agent has
no MCP and cannot look anything up.

#### Scenario: Prompt stands alone

- GIVEN the generated API system prompt
- WHEN it is inspected
- THEN it contains no image markup
- AND it references no MCP tool name
- AND it includes the grammar, op catalogue, and pitfalls required to author source.

#### Scenario: Prompt stays within budget

- GIVEN the configured prompt size ceiling
- WHEN the API system prompt is generated
- THEN its size is at or below the ceiling
- AND the build fails if the distilled prompt overflows.

### Requirement: The op catalogue is injected, never hand-written

The system SHALL build the prompt's op catalogue from `surface-reference` (the
op set derived from code), not from a hand-maintained list, so adding or removing
an op updates the prompt automatically.

#### Scenario: A new op appears without editing the prompt

- GIVEN a newly added op present in `surface-reference`
- WHEN the API system prompt is generated
- THEN the op appears in the prompt's op catalogue
- AND no prompt source was hand-edited to add it.

### Requirement: Artifacts may not drift from the op set

The system SHALL verify that the op set named in the generated prompt, the
`surface-reference` op set, and the book appendix op list agree, and fail the
build when they diverge.

#### Scenario: A stale op mention fails the build

- GIVEN an op renamed or removed in code
- WHEN the drift check runs
- THEN a remaining mention of the old op in the prompt or book appendix fails the check.

### Requirement: The committed prompt cannot go stale

The system SHALL commit the generated API system prompt as a reviewable file and
verify in CI that regenerating it from source yields no diff, so the checked-in
prompt always matches its source.

#### Scenario: Stale committed prompt fails CI

- GIVEN the committed API system prompt
- WHEN the language source changes but the prompt is not regenerated
- THEN the CI freshness check regenerates it and detects the diff
- AND the build fails until the committed prompt is refreshed.

### Requirement: MCP and API share one language body

The system SHALL serve the MCP language guide / technical-system-prompt and the
API system prompt from one shared builder so their language rules are identical.

#### Scenario: MCP and API language bodies match

- GIVEN the shared language-reference builder
- WHEN the MCP-served language body and the API prompt language body are compared
- THEN they are identical
- AND neither path carries a forked copy of the rules.
