# Ecky Agent Language Gaps Plan

## Purpose

Fix gaps visible during real CAD authoring:

- weak dimensional constraints
- weak layout/debug visibility
- repeated geometry written as copy-paste instances
- agent still forced to reason from source text too often

Target: agent can ask model why geometry is where it is, patch intent, validate constraints, preview through MCP, then commit. No direct DB writes.

## Current Failure Mode

Trap model exposed same pattern:

- dimensions encoded as ad hoc arithmetic like `(+ holder_w 12)` and `(- holder_h 7)`
- no named relation explaining why 12 or 7 exists
- no first-class debug overlay for bbox, axes, dimensions, clearances, constraints
- repeated shelves, ribs, clips, doors, corridors authored as separate bindings
- MCP has AST tools, but agent workflow still falls back to full source reads for shape-level reasoning

## Slice 1: Named Constraint Values

Goal: stop magic offsets.

### Tasks

- Add style rule: derived dimensions get named bindings, not inline arithmetic.
- Add authoring lint for repeated numeric deltas used in geometry expressions.
- Add constraint metadata surface:
  - `constraintId`
  - `label`
  - `kind`
  - `sourceStableNodeKey`
  - `dependsOnParamKeys`
  - `affectsStableNodeKeys`
- Extend `ecky_constraints_validate` response with named relations, not only pass/fail.
- Teach prompt/authoring guide: prefer `holder_margin_x`, `holder_margin_y`, `door_clearance`, `lid_capture_depth`.

### Acceptance

- Given source has `(+ holder_w 12)` in shape placement
- When constraint lint runs
- Then response proposes named binding `holder_margin_x`
- And MCP constraint validation shows relation owner and dependent geometry.

### Tests

- Rust unit: expression scan flags anonymous numeric delta in shape-relevant expressions.
- MCP handler test: `ecky_constraints_validate` returns relation metadata.
- Fixture: woodlouse trap source with holder margins.

## Slice 2: Layout Debug Primitives

Goal: agent and user can inspect structure before changing geometry.

### DSL Surface

```scheme
(debug
  (show bbox)
  (show axes)
  (show dimensions)
  (show constraints))
```

### Runtime Surface

- Debug primitives must not export into production STL/STEP by default.
- Preview can include debug overlays when requested by MCP/UI.
- Overlay output must be separate manifest section:
  - `debugOverlays`
  - `bbox`
  - `axisTripods`
  - `dimensionLines`
  - `constraintLabels`

### Tasks

- Parse `debug` forms in Ecky source.
- Lower debug overlays into manifest only, not solid geometry.
- Add MCP `target_detail_get(section="debugOverlays")`.
- Add viewport debug toggle that reads manifest overlays.
- Keep Tactical Midnight theme; no floating tutorial text.

### Acceptance

- Given trap model has `show dimensions`
- When MCP preview renders
- Then artifact manifest includes length/width/height dimension lines
- And exported STL remains unchanged.

### Tests

- Rust unit: debug forms parse and survive compile metadata.
- Render service test: debug overlays absent from STL assets.
- Playwright: toggle debug overlays and see bbox/dimension layer.

## Slice 3: Instances And Repeat Ergonomics

Goal: repeated features stay one intent, many placements.

Existing lowerers already know `repeat`, `repeat-union`, `repeat-compound`, `repeat-pick`. Missing part is ergonomic authored instances and manifest provenance.

### DSL Surface

```scheme
(instance door-left
  :of door
  :at (- half_l door_offset) corridor_y 0
  :params ((side "left")))

(repeat (i door_count)
  (instance door
    :at (door-x i) (door-y i) 0))
```

### Tasks

- Add `instance` as source-level semantic form or macro lowering to existing repeat/core.
- Preserve instance identity in Core AST:
  - `instanceId`
  - `prototypeStableNodeKey`
  - `placementExprStableNodeKey`
  - `paramOverrides`
- Export instance provenance into manifest selection targets.
- Add dependency graph:
  - prototype param -> all instances
  - placement param -> specific instance placements
- Add AST patch operation for repeated instance bodies.

### Acceptance

- Given four sliding doors are generated by one prototype and repeat
- When agent changes door thickness
- Then all doors update
- When agent changes one placement override
- Then only selected instance moves
- Manifest still maps clicked door back to prototype and instance.

### Tests

- Rust unit: `instance` macro expands to valid Core while retaining metadata.
- Lowering parity: FreeCAD and build123d emit same object count/bounds for repeated doors.
- MCP AST test: selected instance reports prototype and override nodes.

## Slice 5: Constraint-Driven Authoring For Physical Models

Goal: physical dimensions remain coherent after parameter edits.

### Required Constraint Kinds

- `clearance`
- `wall_min`
- `fit_inside`
- `align`
- `symmetric_pair`
- `printable_overhang`
- `export_invariant`

### Tasks

- Encode constraints in model manifest.
- Validate before render when possible.
- Validate after render using bbox/topology evidence.
- Surface raw violations to MCP and UI:
  - constraint id
  - expected relation
  - actual measured value
  - source stable keys
  - affected params
- Add repair proposals, but do not auto-apply.

### Acceptance

- Given lid height changes
- Then rail capture, door clearance, and lid skirt constraints validate
- If invalid, render preview rejects or returns explicit warning before commit.

### Tests

- Unit: constraint evaluator for numeric relation and bbox relation.
- MCP: invalid clearance returns exact raw violation.
- Playwright: param apply shows violation, not generic render error.

## Slice 6: Authoring Policy And Prompt Contract

Goal: stop agent regressions.

### Rules

- New CAD model with repeated structures must use `repeat` or `instance`.
- New physical fit relation must have named constraint or named binding.
- Debug overlays are allowed during preview, forbidden in production export geometry.
- MCP first:
  - inspect
  - validate
  - preview
  - commit
- No SQLite writes.
- No full macro rewrite when AST patch can express the edit.

### Anonymous Offsets: Bad vs Good

Bad (anonymous fit offset in placement):

```scheme
(let ((holder_w 72)
      (door 12))
  (place (location (plane :origin (list (+ holder_w 12) 0 0)))
         door))
```

Good (named binding + named fit constraint):

```scheme
(let* ((holder_w 72)
       (holder_margin_x 12)
       (door_x (+ holder_w holder_margin_x)))
  (constraints
    (fit_inside :id "door-holder-fit" :lhs door_x :rhs (+ holder_w holder_margin_x)))
  (place (location (plane :origin (list door_x 0 0)))
         door))
```

### Tasks

- Update AGENTS.md project section.
- Update MCP authoring card text.
- Add lint warning in app bubble for anonymous geometry offsets.
- Add docs snippet with good/bad examples.

### Acceptance

- Agent prompt tells model to use `shapeGraph`, `ecky_ast_*`, and `ecky_constraints_validate`.
- Bad source pattern produces actionable lint.
- User can see why parameter edit is unsafe before print.

## Implementation Order

1. Constraint naming lint and MCP relation response.
2. Shape graph packet from existing Core AST and manifest.
3. Debug overlays as manifest-only data.
4. Instance/prototype metadata on top of existing repeat lowerers.
5. Constraint evaluator with physical relation kinds.
6. UI overlay toggle and violation surfacing.
7. Authoring policy updates.

## Stop Conditions

- Any path writes app SQLite directly.
- Debug overlay enters exported STL/STEP as solid geometry.
- Agent must read entire source for simple `which shape controls this?`.
- Constraint violation renders as generic error.
- Instance edit loses per-instance identity.

## Done Definition

- Woodlouse trap can be represented as:
  - two side corridor instances
  - one central food hall
  - one sliding door prototype repeated four times
  - one lid with named lip/skirt constraints
- Agent can inspect this through MCP shape graph.
- Agent can patch one param or one repeated prototype through AST tool.

- Debug overlay shows bbox, axes, dimensions, and constraints without contaminating export.
